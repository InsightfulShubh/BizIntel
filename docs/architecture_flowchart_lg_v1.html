❌ v3 — Linear chain.py
Monolithic analyze() method
All logic crammed into a single 80-line function. No retries — if confidence is low, you get one shot. Branching via if/elif. Hard to test, hard to extend.
def analyze(query, analysis_type): expanded = expand(query) # step 1 docs = retrieve(expanded) # step 2 if confidence == "none": # step 3 return REFUSAL elif confidence == "low": answer = llm(prompt) return DISCLAIMER + answer else: answer = llm(prompt) # step 4 return answer # ← no retry, no validation # ← no corrective RAG loop

✅ LG v1 — StateGraph
12 Nodes · 2 Conditional Edges · Corrective RAG
Each step is an isolated node with a single responsibility. Conditional edges handle routing. Rewrite loop enables corrective RAG — retry with rewritten query.
graph = StateGraph(PrivateState) graph.add_node("classify", classify) graph.add_node("expand_query", expand) graph.add_node("retrieve", retrieve) graph.add_node("confidence_gate", gate) graph.add_node("generate_similar", gen_sim) # ... 5 type-specific generate nodes graph.add_node("validate", validate) graph.add_node("rewrite", rewrite) # ← RETRY! graph.add_node("refuse", refuse) # Conditional routing + retry loop graph.add_edge("rewrite", "expand_query")

📤 Output Schema
OutputState(BaseModel)
state/output.py
What graph.invoke() returns. Only these fields are visible to the caller.
class OutputState(BaseModel): answer: str = "" analysis_type: str = "" source_docs: list[dict] = [] confidence: str = "" best_score: float = 0.0
graph output
Design pattern: Multiple inheritance — PrivateState(InputState, OutputState). This gives LangGraph schema filtering: the caller sends InputState, graph works with PrivateState, result is filtered to OutputState. Internal fields like expanded_query, retry_count never leak out.

Conditional Edges — Routing Logic
graph/edges.py
CONDITIONAL EDGE #1
route_after_confidence(state)
After: confidence_gate
Routes to the correct generate node based on analysis_type, or to rewrite/refuse based on confidence.
if confidence in ("high", "low"): → generate_{analysis_type} # e.g. "generate_swot" elif confidence == "none": if retry_count < MAX_RETRIES: → "rewrite" else: → "refuse" → __end__
CONDITIONAL EDGE #2
route_after_validate(state)
After: validate
Decides whether the answer is good enough, or needs a corrective rewrite.
if validation_check == True: → __end__ # pass! elif confidence == "low": → __end__ # best-effort elif retry_count < MAX_RETRIES: → "rewrite" # try again else: → __end__ # exhausted

Design Patterns — Closure DI · Factory · DRY Helper
Pattern 1
Closure-Based Dependency Injection
build_graph() creates closures that capture retriever and llm_client. State stays pure Pydantic — no ORM, no singletons.
def build_graph(retriever, llm_client): classify = make_classify_node(llm_client) retrieve = make_retrieve_node(retriever) graph.add_node("classify", classify) # llm_client captured in closure # retriever captured in closure
Closure DI
Pattern 2
Factory Functions (make_*_node)
Each LLM/retriever node has a make_*_node(dep) factory that returns a closure. Pure logic nodes (confidence_gate) are plain functions — no factory needed.
def make_classify_node(llm_client): def classify_node(state) -> dict: # uses llm_client from closure return {"analysis_type": "swot"} return classify_node # vs. pure logic (no factory): def confidence_gate_node(state) -> dict: return {"confidence": "high"}
Factory
Pattern 3
DRY Helper (_generate_base.py)
All 5 generate nodes call run_generation() with a different analysis_type. Shared prompt building, LLM call, and disclaimer logic.
# _generate_base.py (not a node!) def run_generation(llm, state, *, analysis_type): prompt = get_prompt(analysis_type, ...) answer = llm.chat(prompt) if confidence == "low": answer = DISCLAIMER + answer return {"answer": answer} # generate_swot.py: def make_generate_swot_node(llm): def node(state): return run_generation( llm, state, analysis_type="swot" ) return node
