Web Fallback Path
When confidence=none and retry_count >= MAX_RETRIES and WEB_SEARCH_ENABLED=True:
web_search (Tavily) → web_review (HITL interrupt) → generate_* → validate → record_turn → END
The graph pauses at web_review until the user sends Command(resume=[approved_indices]).

Human-in-the-Loop — interrupt() + Command(resume=...)
HITL Pattern
graph/nodes/web_review.py
How HITL Works
Pause → Review → Resume
The web_review node calls interrupt(payload) which pauses the graph. The payload is sent to the Streamlit UI. The user reviews web results and selects which to keep. Streamlit sends Command(resume=[0, 2, 4]) to resume the graph. The node filters source_docs to the approved subset.
# In web_review_node: approved_indices = interrupt({ "type": "web_review", "message": "Select which results to use:", "results": [ {"index": 0, "title": "...", "url": "..."}, {"index": 1, "title": "...", "url": "..."}, ] }) # Graph pauses here ↑ # User reviews in Streamlit UI # User clicks "Approve Selected" # Graph resumes here ↓ # approved_indices = [0, 2] approved_docs = [doc for i, doc in enumerate(source_docs) if i in approved_indices] return {"source_docs": approved_docs}
Streamlit Side
_render_hitl_review() + _resume_graph()
When _stream_graph() detects a pending interrupt, it saves the interrupt data to st.session_state and reruns. The HITL UI renders checkboxes for each web result. On "Approve", it calls graph.stream(Command(resume=selected)).
# In streamlit_app.py: # 1. Detect interrupt graph_state = graph.get_state(config) if pending_interrupts: st.session_state._hitl_pending = True st.session_state._hitl_data = data st.rerun() # 2. Render review UI (checkboxes) def _render_hitl_review(): for result in data["results"]: checked = st.checkbox(title) if st.button("✅ Approve"): st.session_state._hitl_resume = indices # 3. Resume graph def _resume_graph(): result = graph.stream( Command(resume=resume_value), config=config, )
Requires checkpointer: The HITL pattern only works with a checkpointer (SQLite in production, MemorySaver in notebook). The checkpointer serializes the graph state at the interrupt point, and restores it when Command(resume=...) is sent.

Persistent Conversation Memory
SQLite · thread_id
Checkpointer
SQLite SqliteSaver
Each Streamlit session gets a unique thread_id. The SQLite checkpointer persists full graph state (including conversation_history) across reruns.
import sqlite3 from langgraph.checkpoint.sqlite \ import SqliteSaver conn = sqlite3.connect( "data/conversations.db", check_same_thread=False) checkpointer = SqliteSaver(conn=conn) graph = build_graph( ..., checkpointer=checkpointer )

Context Injection
format_history_context()
The classify node formats recent turns (up to MEMORY_WINDOW=6) and injects them into the LLM prompt. This resolves coreferences like "Now do a SWOT for them".
# graph/utils/history.py def format_history_context(history): recent = history[-MEMORY_WINDOW:] lines = [] for turn in recent: role = turn["role"] content = turn["content"][:200] lines.append(f"{role}: {content}") return "\n".join(lines) # Used in classify_node: history = format_history_context( state.conversation_history )

Design Patterns
Pattern
Human-in-the-Loop (HITL)
Uses LangGraph interrupt() to pause execution at a specific node. The UI collects user input, then Command(resume=value) resumes from exactly where it paused.
HITL interrupt + Command
Pattern
Append-Only Reducer
Annotated[list[dict], operator.add] — LangGraph merges state by appending new elements rather than replacing. Critical for conversation history that accumulates across graph invocations.
State Reducer operator.add
Pattern
Checkpointed State Machine
SQLite checkpointer + thread_id = full state persistence. Conversations survive Streamlit reruns, page refreshes, and even server restarts. Same pattern enables HITL resume.
Persistence SqliteSaver
